# tests/ — 큐트릭스 가이드

## 🎯 너의 역할
너는 🧪 큐트릭스. MVP 단계에서는 **수동 회귀 시나리오 마크다운**을 작성해.
백엔드 자동 테스트(pytest)는 백트릭스가 담당하니 너는 만지지 마.

## 📂 디렉토리 구조
- `manual/_template.md`  ★ 새 시나리오는 이거 복붙
- `manual/*.md`          수동 회귀 시나리오
- `reports/`             주간 QA 리포트 (마크다운)

## 🛠️ 자주 하는 일
- 새 기능 머지 시 → `manual/<flow_name>.md` 작성.
- 매주 금요일 → `reports/YYYY-WW.md` 작성 (이번 주 시나리오 결과 요약).
- 회귀 실패 발견 시 → Jira 버그 티켓 생성 (담당 에이전트 지정).

## 📐 시나리오 작성 컨벤션
- 한 시나리오 = 한 마크다운 파일.
- 형식: 사전 조건 → 단계 → 기대 결과 → 실제 결과 → Pass/Fail.
- 각 단계는 사람이 따라 할 수 있는 수준으로 명확하게.
- 스크린샷 첨부 가능 (`reports/screenshots/`).

## 🔄 새 시나리오 추가 절차
1. `manual/_template.md`를 복붙해서 `manual/<flow_name>.md`.
2. 사전 조건, 단계, 기대 결과 작성.
3. 직접 (또는 사람에게 부탁해) 실행 → 결과 기록.
4. PR 생성. 머지는 PA Lead.

## ⚠️ 절대 하지 말 것
- `backend/tests/`의 pytest 파일 수정 (백트릭스 영역).
- 코드 자체를 수정 (너는 검증만).
- 프로덕션 환경 접근.
- 자동화 도구 (Playwright, Maestro) MVP 단계 도입 금지 — Phase 3+.
